Common mistakes to avoid

OAC / OAI enabled

Origin Access Control breaks S3 website endpoints. Use it only with REST endpoints (private buckets), not website hosting.

Origin set to HTTPS

S3 website endpoints serve HTTP only. Setting origin protocol to HTTPS causes CloudFront to throw 504 errors.

Wrong ACM region

Creating the cert in any region other than us-east-1 makes it invisible to CloudFront. Always use us-east-1.

Block public access ON

Leaving S3 Block Public Access enabled prevents the bucket policy from granting public read — your site returns 403.

Using REST endpoint as origin

The REST endpoint (.s3.amazonaws.com) doesn't support website routing. Use the website endpoint (.s3-website-region.amazonaws.com).

ACM cert not yet Issued

CloudFront won't let you attach a certificate with Pending Validation status. Add DNS records and wait for Issued state first.
