High-Level Architecture

DeliverX is structured as a multi-tier, event-driven application. The frontend (React Native + Next.js web) communicates via a GraphQL/REST API Gateway. Core services are decoupled microservices backed by PostgreSQL, Redis, and MongoDB. Real-time delivery tracking uses WebSockets and Socket.io.

authMiddleware.ts TypeScript 
export const requireRole = (...roles: Role[]) => {
  return (req: Request, res: Response, next: NextFunction) => {
    const token = req.headers.authorization?.split(' ')[1];
    if (!token) return res.status(401).json({ error: 'Unauthorized' });

    try {
      const payload = verifyJWT(token) as JWTPayload;
      if (!roles.includes(payload.role)) {
        return res.status(403).json({ error: 'Forbidden' });
      }
      req.user = payload;
      next();
    } catch {
      res.status(401).json({ error: 'Token invalid or expired' });
    }
  };
};

Place Order — order.controller.ts TypeScript 
export const placeOrder = async (req: AuthRequest, res: Response) => {
  const { restaurantId, items, addressId, paymentMethod, notes } = req.body;

  // 1. Validate restaurant is open & items exist
  const restaurant = await getRestaurantOrThrow(restaurantId);
  const validatedItems = await validateMenuItems(restaurantId, items);

  // 2. Calculate pricing (subtotal + fees + tax)
  const pricing = await pricingService.calculate({
    items: validatedItems, addressId, restaurant
  });

  // 3. Create order in DB (PENDING)
  const order = await orderRepository.create({
    customerId: req.user.id, restaurantId, items: validatedItems,
    addressId, notes, ...pricing, status: 'PENDING'
  });

  // 4. Initiate payment
  const paymentIntent = await paymentService.createIntent({
    orderId: order.id, amount: pricing.total, method: paymentMethod
  });

  // 5. Publish event → notification service
  await eventBus.publish('order.created', { order, restaurant });

  res.status(201).json({ order, paymentIntent });
};

tracking.gateway.ts TypeScript 
io.on('connection', (socket) => {
  // Customer joins order room
  socket.on('order:join', ({ orderId }) => {
    socket.join(`order:${orderId}`);
  });

  // Driver pushes GPS every 3s
  socket.on('driver:location', async (data) => {
    await redis.geoAdd('drivers', {
      longitude: data.lng,
      latitude: data.lat,
      member: data.driverId
    });
    // Broadcast to customer watching order
    io.to(`order:${data.orderId}`)
      .emit('driver:location', data);
  });
});

State Management
Zustand for global client state. React Query for server state + caching. No Redux needed.
useCartStore.ts — Zustand TypeScript 
interface CartStore {
  restaurantId: string | null;
  items: CartItem[];
  addItem: (item: CartItem) => void;
  removeItem: (id: string) => void;
  clearCart: () => void;
  getTotal: () => number;
}

export const useCartStore = create<CartStore>()(
  persist(
    (set, get) => ({
      restaurantId: null,
      items: [],
      addItem: (item) => set((s) => ({
        items: [...s.items, item],
        restaurantId: item.restaurantId
      })),
      removeItem: (id) => set((s) => ({
        items: s.items.filter((i) => i.id !== id)
      })),
      clearCart: () => set({ items: [], restaurantId: null }),
      getTotal: () => get().items.reduce((sum, i) => sum + i.price * i.qty, 0),
    }),
    { name: 'cart-storage' }
  )
);

Dispatch Algorithm

Order Confirmed
Merchant accepts order, dispatch service activates
Find Nearby Drivers
GEORADIUS query: online drivers within 3km of restaurant
Score & Rank
Score = distance × 0.6 + rating × 0.3 + acceptance rate × 0.1
Dispatch Offer
Send offer to top driver; 30s timeout to accept
Fallback
If rejected/timeout → try next driver; widen radius by 1km

Security
Defense in depth: TLS everywhere, RBAC, input validation, rate limiting, WAF, and regular audits.

Security Checklist

  • ✅ All traffic HTTPS only (TLS 1.3)
  • ✅ JWT with short expiry (15 min access tokens)
  • ✅ Refresh token rotation + revocation
  • ✅ bcrypt for any stored passwords (rounds=12)
  • ✅ Input validation on every endpoint (Zod)
  • ✅ SQL injection prevention (Prisma parameterized)
  • ✅ XSS prevention (Helmet.js, CSP headers)
  • ✅ CORS whitelist (only known origins)
  • ✅ Rate limiting per IP + per user (Redis)
  • ✅ AWS WAF (OWASP rule set)
  • ✅ Secrets in AWS Secrets Manager (never in code)
  • ✅ GDPR: data export, deletion, consent tracking
  • ✅ PCI DSS: no raw card data stored (Stripe tokenizes)
  • ✅ Audit logs for admin actions
  • ✅ Snyk dependency vulnerability scanning in CI
  • ✅ SAST (SonarQube) in pipeline
  • ✅ Penetration testing quarterly
🔴 Never store raw payment card numbers. Use Stripe/Razorpay tokens only. All card operations happen on their PCI-compliant servers.
✅ All database connections are restricted to the VPC. No public RDS/Redis endpoints.
⚠️ Driver GPS data is considered sensitive. Only the matched customer can query a driver's location during their active order.
ℹ️ Phone numbers are hashed before indexing in Elasticsearch to prevent leakage in search service logs.

Testing Strategy
Pyramid-based: unit → integration → E2E. 80% coverage minimum. Automated in CI on every commit.
🔬 Unit Tests — Jest
Pure functions, service logic, utility helpers. Mock all external dependencies. Run in <30s. Target: >80% coverage per service.
🔗 Integration Tests
API endpoints with real DB (Docker test containers). Supertest for HTTP. Validates full request/response cycle including DB writes.
🎭 E2E Tests — Playwright
Full user flows on web apps. Critical paths: login → order → track → review. Runs on staging before production deploy.
