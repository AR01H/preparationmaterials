Install AWS CLI v2
One-time install on your machine
PowerShell / cmd — Windows
# Option 1: Download MSI installer (recommended)
# Go to: https://awscli.amazonaws.com/AWSCLIV2.msi
# Double-click and install

# Option 2: via winget (Windows Package Manager)
winget install Amazon.AWSCLI

# Option 3: via Chocolatey
choco install awscli

# Verify install (open new terminal after install)
aws --version
# should print: aws-cli/2.x.x Windows/...
terminal — Linux (Ubuntu/Debian/Amazon Linux)
# Download and install AWS CLI v2
curl "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o "awscliv2.zip"
unzip awscliv2.zip
sudo ./aws/install

# Verify
aws --version
# should print: aws-cli/2.x.x Linux/...

# For ARM (Raspberry Pi / AWS Graviton):
curl "https://awscli.amazonaws.com/awscli-exe-linux-aarch64.zip" -o "awscliv2.zip"

Create IAM User & Access Keys in AWS Console
Do this in AWS website before running any commands
AWS Console — steps to follow
1. Login to AWS Console → https://console.aws.amazon.com
2. Search for "IAM" → Click IAM
3. Left sidebar → Users → Create User
4. Username: give a name (e.g. "dev-user")
5. Permissions: Attach policies directly → choose what you need
   (e.g. AmazonS3FullAccess, AdministratorAccess, etc.)
6. After user is created → click the user name
7. Tab: "Security credentials" → Create access key
8. Use case: "Command Line Interface (CLI)"
9. Copy the two values — you will need them next:

Access Key ID:     AKIAIOSFODNN7EXAMPLE
Secret Access Key: wJalrXUtnFEMI/K7MDENG/bPxRfiCYEXAMPLEKEY

# ⚠️ Save these NOW — Secret Key is shown only ONCE
3
Configure AWS Credentials
For default + multiple profiles (from your notes)
All credential files live in ~/.aws/ on Linux and C:\Users\YOU\.aws\ on Windows. Same structure on both OS.
Windows — aws configure
# Quick setup — default profile
aws configure

# It will ask 4 questions:
AWS Access Key ID: AKIAIOSFODNN7EXAMPLE
AWS Secret Access Key: your-secret-key
Default region name: us-east-1
Default output format: json

# For secondary profile:
aws configure --profile secondary
# fills secondary profile credentials
Linux — same commands
# Identical commands on Linux
aws configure

AWS Access Key ID: AKIAIOSFODNN7EXAMPLE
AWS Secret Access Key: your-secret-key
Default region name: ap-south-1
Default output format: json

# For secondary profile:
aws configure --profile secondary
~/.aws/credentials (Linux)  |  C:\Users\YOU\.aws\credentials (Windows) 
# Default profile
[default]
aws_access_key_id = AKIAIOSFODNN7EXAMPLE
aws_secret_access_key = wJalrXUtnFEMI/K7MDENG

# Secondary profile
[secondary]
aws_access_key_id = AKIAI2NDKEY7EXAMPLE
aws_secret_access_key = 2ndSecretKeyHere
~/.aws/config (Linux)  |  C:\Users\YOU\.aws\config (Windows) 
# Default profile
[default]
region = us-east-1
output = json

# Secondary profile (note: "profile" prefix required)
[profile secondary]
region = us-east-2
output = json

Test Connection & Use Profiles
Verify your credentials work and switch between accounts
terminal — test and switch profiles (same on Windows & Linux)
# Test default profile
aws sts get-caller-identity
# shows: Account ID, UserID, ARN

# Test secondary profile
aws sts get-caller-identity --profile secondary

# List S3 buckets — default account
aws s3 ls

# List S3 buckets — secondary account
aws s3 ls --profile secondary

# Set default profile for current session
# Linux/Mac:
export AWS_PROFILE=secondary

# Windows PowerShell:
$env:AWS_PROFILE = "secondary"

# Windows CMD:
set AWS_PROFILE=secondary

Use AWS SDK in Code
Node.js, Python — same on Windows & Linux
Node.js — install & use AWS SDK v3
# Install SDK
npm install @aws-sdk/client-s3
npm install @aws-sdk/client-dynamodb

# ── app.js ──────────────────────────────────
const { S3Client, ListBucketsCommand } = require('@aws-sdk/client-s3');

// Default profile (reads from ~/.aws/credentials)
const client = new S3Client({ region: 'us-east-1' });

// Secondary profile
const { fromIni } = require('@aws-sdk/credential-providers');
const client2 = new S3Client({
  region: 'us-east-2',
  credentials: fromIni({ profile: 'secondary' })
});

// List buckets
async function listBuckets() {
  const data = await client.send(new ListBucketsCommand({}));
  console.log(data.Buckets);
}
listBuckets();
Python — install boto3 & use
# Install boto3
pip install boto3            # Linux
pip install boto3            # Windows (same)

# ── app.py ──────────────────────────────────
import boto3

# Default profile
s3 = boto3.client('s3', region_name='us-east-1')

# Secondary profile
session = boto3.Session(profile_name='secondary')
s3_secondary = session.client('s3', region_name='us-east-2')

# List buckets
response = s3.list_buckets()
for bucket in response['Buckets']:
    print(bucket['Name'])

# Upload file to S3
s3.upload_file('local-file.txt', 'my-bucket', 's3-key.txt')

AWS Profiles in Terraform
Use multiple AWS accounts from your credentials file
main.tf — default + secondary profile
# Default provider — no profile needed
provider "aws" {
  region = "us-east-1"
}

# Secondary provider — uses "secondary" profile from ~/.aws/credentials
provider "aws" {
  alias   = "secondary"
  region  = "us-east-2"
  profile = "secondary"
}

# Use secondary provider in a resource
resource "aws_s3_bucket" "backup" {
  provider = aws.secondary
  bucket   = "my-backup-bucket"
}
