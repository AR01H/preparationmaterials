Universal Cost-Saving Principles

These strategies apply across every platform — cloud or on-premises. Master these first before diving into platform-specific tips.

⚡ Quick Wins — Do These This Week
Each takes under 2 hours and typically saves thousands/month
01
Turn Off Idle Resources

Dev/staging servers running nights & weekends = wasted money. Schedule auto-stop scripts. Saves 60%+ of dev environment costs.

02
Delete Unattached Storage

Orphaned disks, snapshots, and old AMIs silently rack up bills. Run a cleanup script monthly. Easy $100–$2000/month savings.

03
Set Billing Alerts

Configure budget alarms at 80% and 100% of expected spend. You can't manage what you can't measure. Free to set up.

04
Tag Every Resource

Without tags, you can't allocate costs to teams or projects. Mandatory tags: environment, team, project, owner. Enables chargeback.

05
Review & Delete Unused LBs

Load balancers with no registered targets still cost money. Often left behind after migrations. $15–40/month each.

06
Compress & Cache Everything

Enable gzip/brotli compression. Use CDN for static assets. Reduce data transfer costs by 60–80% and improve performance.

EC2 Scheduler — Auto Stop Dev/Staging
Dev environments running 24/7 is pure waste. Auto-schedule stop/start.
✦ 60–70% dev environment savings
Without scheduler (168 hrs/week)$280/mo
With scheduler (50 hrs/week)$83/mo

Use AWS Instance Scheduler (free CloudFormation template) or EC2 Auto Scaling schedules. Can save $2,000+/month on a 10-server dev environment.

ec2-scheduler-lambda.py
# Lambda function: auto-stop dev instances at 7PM, start at 8AM
import boto3
ec2 = boto3.client('ec2', region_name='ap-south-1')

def lambda_handler(event, context):
  action = event.get('action', 'stop')

  # Find all dev instances by tag
  response = ec2.describe_instances(Filters=[
    {'Name': 'tag:Environment', 'Values': ['dev', 'staging']},
    {'Name': 'instance-state-name', 'Values': ['running']}
  ])
  ids = [i['InstanceId'] for r in response['Reservations'] for i in r['Instances']]

  if ids:
    if action == 'stop': ec2.stop_instances(InstanceIds=ids)
    if action == 'start': ec2.start_instances(InstanceIds=ids)
    print(f"{action}ped {len(ids)} instances")

# EventBridge Rule: cron(0 13 ? * MON-FRI *) = 7PM IST stop
# EventBridge Rule: cron(0 2 ? * MON-FRI *) = 8AM IST start

BigQuery Cost Optimization
BigQuery charges per TB scanned. One bad query = big bill.
✦ 50–90% query cost savings
  • ✓
    SELECT specific columns NEVER *: BQ charges per column scanned. SELECT * on 10TB table when you need 2 columns = 5x overbilling
  • ✓
    Partition & cluster tables: Partition by date, cluster by frequently filtered columns. Reduces scanned data by 80–99%
  • ✓
    Use materialized views: Pre-aggregate frequently run reports. Scans cached result not raw data
  • ✓
    Set query cost caps: max_bytes_billed prevents runaway queries from billing you
  • ✓
    BigQuery BI Engine: For dashboards, pay flat rate for in-memory cache vs per-query cost
bigquery-save.sql
-- ❌ BAD: Scans entire table
SELECT * FROM `project.dataset.events`
WHERE user_id = 'abc';

-- ✅ GOOD: Partition filter + specific columns
SELECT user_id, event_type, created_at
FROM `project.dataset.events`
WHERE DATE(created_at) = '2024-01-15' -- partition prune
  AND user_id = 'abc';

-- Set max bytes billed per query (safety cap)
-- In API: set maximumBytesBilled = 10737418240 (10GB)

resources:
  requests: # Scheduler uses this for bin-packing
    cpu: "250m" # 0.25 cores minimum

    cpu: "500m" # 0.5 cores max burst

kind: VerticalPodAutoscaler

  updatePolicy:
    updateMode: "Off" # Recommend only, don't auto-change
Strategic Cloud Bursting (Hybrid)
Keep baseline on-prem; burst expensive peaks to cloud. Best of both worlds.
✦ 40–60% vs pure cloud
  • ✓
    Cloud bursting pattern: Run 80% load on-prem; use cloud spot/preemptible for the remaining 20% peak spikes
  • ✓
    Offload ML training: Keep inference on-prem (low latency), burst training to GCP TPUs or AWS P-instances
  • ✓
    Backup to cloud: On-prem tape backup is expensive (maintenance, media, labor). S3 Glacier is $0.004/GB/month
  • ✓
    DR in cloud: Maintain DR site in cloud (warm standby) instead of second DC. Saves 60–80% on DR costs
  • ✓
    Static assets to CDN: Offload images/video to cloud CDN. Remove this traffic from your on-prem bandwidth

On-Prem vs Cloud Breakeven: On-prem typically breaks even vs cloud at 3–5 years for stable, predictable workloads. Variable/seasonal workloads almost always favor cloud. Run a TCO analysis with: hardware, power, cooling, datacenter rent, staff salary, maintenance contracts.
