How Docker Works for You
Your Code
Stays on your Windows folder (C:\myproject)
Volume Mount
Docker maps your folder into the container automatically
Container Runs It
PHP/Python/MySQL run inside Docker — not on Windows
Key idea: You edit code in VS Code on Windows. Docker runs it. You never install PHP, Python, or MySQL on your machine.

Full Stack Setup — All Services Together

Create a file named docker-compose.yml in your project folder C:\myproject\

version: '3.8'

services:

  # ─── MySQL Database ───────────────────────────
  mysql:
    image: mysql:8.0
    container_name: my_mysql
    ports:
      - "3306:3306"        # Access on localhost:3306
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: mydb
      MYSQL_USER: user
      MYSQL_PASSWORD: password
    volumes:
      - ./mysql-data:/var/lib/mysql          # Stores DB data
      - ./dump.sql:/docker-entrypoint-initdb.d/dump.sql  # Auto-import dump

  # ─── PHP with Apache ──────────────────────────
  php:
    build: ./php               # Custom Dockerfile with Xdebug
    container_name: my_php
    ports:
      - "8080:80"              # Visit http://localhost:8080
    volumes:
      - ./php-code:/var/www/html   # Your PHP files here
    environment:
      XDEBUG_MODE: debug
      XDEBUG_CONFIG: client_host=host.docker.internal client_port=9003
    depends_on:
      - mysql

  # ─── Python ───────────────────────────────────
  python:
    image: python:3.11
    container_name: my_python
    ports:
      - "5000:5000"            # Flask or FastAPI
      - "5678:5678"            # Debugpy port for VS Code
    volumes:
      - ./python-code:/app
    working_dir: /app
    command: pip install -r requirements.txt && python -m debugpy --listen 0.0.0.0:5678 --wait-for-client app.py
    depends_on:
      - mysql

  # ─── Node.js ──────────────────────────────────
  nodejs:
    image: node:20
    container_name: my_node
    ports:
      - "3000:3000"            # Visit http://localhost:3000
      - "9229:9229"            # Node.js debug port
    volumes:
      - ./node-code:/app
    working_dir: /app
    command: sh -c "npm install && node --inspect=0.0.0.0:9229 index.js"

  # ─── Drupal ───────────────────────────────────
  drupal:
    image: drupal:10-apache
    container_name: my_drupal
    ports:
      - "8090:80"              # Visit http://localhost:8090
    volumes:
      - ./drupal-modules:/var/www/html/modules/custom
      - ./drupal-themes:/var/www/html/themes/custom
    depends_on:
      - mysql

  # ─── phpMyAdmin (optional, manage DB via browser) ─
  phpmyadmin:
    image: phpmyadmin:latest
    container_name: my_phpmyadmin
    ports:
      - "8888:80"              # Visit http://localhost:8888
    environment:
      PMA_HOST: mysql
      PMA_USER: root
      PMA_PASSWORD: root
    depends_on:
      - mysql

For Drupal, during setup in browser use these DB settings:

Database hostmysql (not localhost!)
Database namemydb
Usernameroot
Passwordroot
Port3306
Inside Docker containers, services talk to each other using the service name (mysql), not localhost.
Start everything:
cd C:\myproject
docker-compose up -d

To run specific services only (even from full compose file):
# Only start PHP and MySQL (ignore other services)
docker-compose up -d php mysql phpmyadmin

# Only start Python
docker-compose up -d python

# Only start Node.js
docker-compose up -d nodejs
PHP Dockerfile (needed for Xdebug)

Create a folder php/ inside your project and add this Dockerfile:

FROM php:8.2-apache

# Install Xdebug for debugging in VS Code
RUN pecl install xdebug && docker-php-ext-enable xdebug

# Install mysqli extension to connect to MySQL
RUN docker-php-ext-install mysqli pdo pdo_mysql

# Xdebug config
RUN echo "xdebug.mode=debug" >> /usr/local/etc/php/conf.d/xdebug.ini \
 && echo "xdebug.start_with_request=yes" >> /usr/local/etc/php/conf.d/xdebug.ini \
 && echo "xdebug.client_host=host.docker.internal" >> /usr/local/etc/php/conf.d/xdebug.ini \
 && echo "xdebug.client_port=9003" >> /usr/local/etc/php/conf.d/xdebug.ini
Folder Structure
C:\myproject\ │ ├── docker-compose.yml <-- main config file ├── dump.sql <-- your MySQL dump from staging │ ├── php/ <-- PHP Dockerfile folder │ └── Dockerfile │ ├── php-code/ <-- Your PHP files (edit in VS Code) │ └── index.php │ ├── python-code/ <-- Your Python files │ ├── app.py │ └── requirements.txt │ ├── node-code/ <-- Your Node.js files │ ├── index.js │ └── package.json │ ├── drupal-modules/ <-- Custom Drupal modules ├── drupal-themes/ <-- Custom Drupal themes │ └── mysql-data/ <-- Auto-created, stores MySQL data
You only edit files in php-code/, python-code/, node-code/ — everything else is Docker managed.
Import MySQL Dump from Staging
Method 1 — Auto import on first start

Put your dump.sql in your project folder. It's auto-imported when MySQL container starts for the first time.

Only works on first start. If MySQL data already exists, it won't re-import.
Method 2 — Manual import anytime
docker exec -i my_mysql mysql -u root -proot mydb < dump.sql

This is just like importing locally — same thing, different tool.

Take a dump FROM the container:
docker exec my_mysql mysqldump -u root -proot mydb > my_backup.sql
VS Code Debugging — Same as Your Current Workflow
Step 1 — Install VS Code Extension

Search for PHP Debug by Xdebug in VS Code extensions panel and install it.

Step 2 — Create launch.json

In VS Code press Ctrl+Shift+P → "Open launch.json" and paste:

{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "PHP Docker Debug",
      "type": "php",
      "request": "launch",
      "port": 9003,
      "pathMappings": {
        "/var/www/html": "${workspaceFolder}/php-code"
      }
    }
  ]
}
Set a breakpoint in VS Code → Open http://localhost:8080 in browser → Debugger pauses just like before!
Step 1 — Install debugpy in container

Add to your requirements.txt:

debugpy
Step 2 — launch.json
{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Python Docker Debug",
      "type": "python",
      "request": "attach",
      "connect": {
        "host": "localhost",
        "port": 5678
      },
      "pathMappings": [
        {
          "localRoot": "${workspaceFolder}/python-code",
          "remoteRoot": "/app"
        }
      ]
    }
  ]
}
Container starts and waits. In VS Code press F5 to attach → Set breakpoints → Same as local debug!
launch.json for Node.js

Node already runs with --inspect flag in the compose file above.

{
  "version": "0.2.0",
  "configurations": [
    {
      "name": "Node.js Docker Debug",
      "type": "node",
      "request": "attach",
      "port": 9229,
      "address": "localhost",
      "localRoot": "${workspaceFolder}/node-code",
      "remoteRoot": "/app",
      "restart": true
    }
  ]
}
Start container → F5 in VS Code → Set breakpoints in your index.js → Works exactly like before!
Useful Commands
Start / Stop
docker-compose up -dStart all containers
docker-compose downStop all containers
docker-compose up -d php mysqlStart only PHP + MySQL
docker-compose stop phpStop only PHP
docker-compose restart phpRestart PHP
Logs & Shell
docker-compose logs phpSee PHP logs
docker-compose logs -f pythonLive Python logs
docker exec -it my_php bashOpen PHP terminal
docker exec -it my_mysql bashOpen MySQL terminal
docker psSee running containers
MySQL
docker exec -i my_mysql mysql -u root -proot mydb < dump.sqlImport dump
docker exec my_mysql mysqldump -u root -proot mydb > backup.sqlExport dump
docker exec -it my_mysql mysql -u root -prootOpen MySQL CLI
Rebuild & Cleanup
docker-compose up -d --buildRebuild after Dockerfile change
docker-compose down -vStop + remove volumes
docker system pruneClean unused Docker data
