Docker Setup
Windows & Linux

Install Docker, write Dockerfile, build images, run containers, use Docker Compose — complete end to end guide.

Windows Linux Docker Engine
1
Install Docker
One-time install — pick your OS
Windows — Docker Desktop
# Step 1: Enable WSL2 (required) — open PowerShell as Admin
wsl --install
# restart computer after this

# Step 2: Download Docker Desktop
# https://docs.docker.com/desktop/install/windows-install/
# Double-click installer → follow steps

# Step 3: Verify (open new terminal)
docker --version
docker ps
# should show: Docker version 24.x.x

# Also works via winget:
winget install Docker.DockerDesktop
Linux — Docker Engine (Ubuntu/Debian)
# Step 1: Remove old versions
sudo apt remove docker docker-engine docker.io containerd runc

# Step 2: Install via official script (easiest)
curl -fsSL https://get.docker.com | sh

# Step 3: Add your user to docker group (no sudo needed)
sudo usermod -aG docker $USER
# log out and log back in after this

# Step 4: Start Docker service
sudo systemctl enable docker
sudo systemctl start docker

# Step 5: Verify
docker --version
docker run hello-world   ← runs test container
2
Write a Dockerfile
Recipe for building your app image — same on Windows & Linux
Dockerfile — Node.js app
# Base image
FROM node:18-alpine

# Set working directory inside container
WORKDIR /app

# Copy package files first (layer caching)
COPY package*.json ./

# Install dependencies
RUN npm install --production

# Copy rest of the code
COPY . .

# Expose port
EXPOSE 3000

# Command to start app
CMD ["node", "index.js"]
Dockerfile — Python/React app
# Python app
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt
COPY . .
EXPOSE 8000
CMD ["python", "app.py"]

# ── React app (multi-stage build) ──
FROM node:18-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm install
COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/build /usr/share/nginx/html
EXPOSE 80
3
Build Image & Run Container
Same commands on Windows & Linux
terminal — build, run, manage containers
# ── Build image ───────────────────────────────
docker build -t my-app .
docker build -t my-app:v1.0 .     ← with version tag

# ── Run container ─────────────────────────────
docker run my-app                  ← runs in foreground
docker run -d my-app               ← runs in background (detached)
docker run -d -p 3000:3000 my-app  ← maps host:container port
docker run -d -p 3000:3000 --name mycontainer my-app

# ── Manage containers ─────────────────────────
docker ps                ← list running containers
docker ps -a             ← list all (including stopped)
docker stop mycontainer
docker start mycontainer
docker rm mycontainer    ← delete container
docker logs mycontainer  ← see logs
docker logs -f mycontainer  ← follow logs live

# ── Enter running container shell ─────────────
docker exec -it mycontainer sh    ← alpine images
docker exec -it mycontainer bash  ← ubuntu-based images

# ── Manage images ─────────────────────────────
docker images            ← list all images
docker rmi my-app        ← delete image
docker system prune      ← remove all unused (images + containers)

Volumes & Environment Variables
Persist data and pass config to containers
Volumes — persist data
# Mount local folder into container
docker run -d \
  -v /home/user/data:/app/data \
  my-app

# Windows path (PowerShell)
docker run -d `
  -v C:\Users\YOU\data:/app/data `
  my-app

# Named volume (Docker manages it)
docker volume create mydata
docker run -d -v mydata:/app/data my-app

# List volumes
docker volume ls
Environment variables
# Pass env vars at runtime
docker run -d \
  -e NODE_ENV=production \
  -e DB_HOST=localhost \
  -e DB_PASS=secret \
  my-app

# Load from .env file
docker run -d --env-file .env my-app

# .env file contents:
NODE_ENV=production
DB_HOST=localhost
DB_PORT=5432
DB_PASS=secret

Docker Compose — Run Multiple Containers
App + DB + Redis together — same on Windows & Linux
docker-compose.yml — full stack example
version: '3.8'

services:

  # Your Node.js app
  app:
    build: .
    ports:
      - "3000:3000"
    environment:
      - NODE_ENV=production
      - DB_HOST=db
      - REDIS_HOST=redis
    volumes:
      - .:/app                    ← live reload during dev
    depends_on:
      - db
      - redis

  # PostgreSQL database
  db:
    image: postgres:15
    environment:
      POSTGRES_DB: myapp
      POSTGRES_USER: admin
      POSTGRES_PASSWORD: secret
    volumes:
      - pgdata:/var/lib/postgresql/data
    ports:
      - "5432:5432"

  # Redis cache
  redis:
    image: redis:7-alpine
    ports:
      - "6379:6379"

volumes:
  pgdata:   ← named volume for DB persistence
terminal — docker compose commands
# Start all services (builds if needed)
docker compose up
docker compose up -d        ← background
docker compose up --build   ← force rebuild images

# Stop all services
docker compose down
docker compose down -v      ← also delete volumes

# View logs
docker compose logs
docker compose logs -f app  ← follow specific service

# Run command in a service
docker compose exec app bash
docker compose exec db psql -U admin myapp
6
Push Image to Docker Hub / ECR
Share your image or deploy to cloud
Docker Hub
# Login to Docker Hub
docker login

# Tag image with your username
docker tag my-app \
  yourusername/my-app:latest

# Push
docker push yourusername/my-app:latest

# Pull anywhere
docker pull yourusername/my-app:latest
AWS ECR (Elastic Container Registry)
# Login to ECR
aws ecr get-login-password \
  --region us-east-1 | \
docker login --username AWS \
  --password-stdin \
  123456789.dkr.ecr.us-east-1.amazonaws.com

# Tag and push
docker tag my-app:latest \
  123456789.dkr.ecr.us-east-1.amazonaws.com/my-app:latest

docker push \
  123456789.dkr.ecr.us-east-1.amazonaws.com/my-app:latest
