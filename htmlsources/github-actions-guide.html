Core Concepts

⚙️

Workflow

A YAML file in .github/workflows/. Defines automation rules — what to do, when to do it, where to run.

🎯

Event / Trigger

What starts a workflow — push, pull_request, schedule, workflow_dispatch, or any GitHub event.

📦

Job

A set of steps that run on the same runner. Jobs run in parallel by default; use needs: for ordering.

🪜

Step

A single task inside a job — either a shell command (run:) or a pre-built action (uses:).

🖥️

Runner

The virtual machine that executes steps. GitHub provides Ubuntu, Windows, macOS runners free-tier.

🔌

Action

A reusable unit of automation. Use marketplace actions (actions/checkout@v4) or write your own.

02

Workflow File Anatomy

Every workflow lives in .github/workflows/<name>.yml

.github/workflows/ci.yml YAML 
# ── 1. Workflow name (shows in GitHub UI) ──────────────────
name: CI Pipeline

# ── 2. Trigger: when does this workflow run? ──────────────
on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

# ── 3. Environment variables (global scope) ───────────────
env:
  NODE_VERSION: '20'
  APP_NAME: my-app

# ── 4. Jobs ───────────────────────────────────────────────
jobs:

  # Job ID (can be anything kebab-case)
  build-and-test:
    name: Build & Test            # Display name
    runs-on: ubuntu-latest         # Runner OS

    # Steps run sequentially inside this job
    steps:

      # uses: pulls a reusable action from marketplace
      - uses: actions/checkout@v4       # clone repo

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: ${{ env.NODE_VERSION }}
          cache: npm                    # auto-cache node_modules

      # run: executes shell commands
      - name: Install dependencies
        run: npm ci

      - name: Run tests
        run: npm test

      - name: Build
        run: npm run build
03

Triggers (on:)

Trigger When it fires Common use
push On any commit pushed CI on every commit
pull_request When a PR is opened/updated Gating PRs before merge
schedule Cron expression Nightly builds, reports
workflow_dispatch Manual trigger from GitHub UI Deploy on demand
workflow_call Called by another workflow Reusable workflows
release When a release is published Publish to npm / PyPI
trigger examples YAML 
# Cron schedule (UTC): every day at 2 AM
on:
  schedule:
    - cron: '0 2 * * *'

# Manual trigger with custom inputs
on:
  workflow_dispatch:
    inputs:
      environment:
        description: 'Deploy to'
        required: true
        default: 'staging'
        type: choice
        options: [staging, production]

# Push only on specific paths
on:
  push:
    paths:
      - 'src/**'
      - 'package.json'
    branches: [main]
04

Jobs, Steps & Contexts

job patterns YAML 
jobs:

  # ── Job sequencing with needs: ─────────────────────
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: npm test

  deploy:
    needs: test                   # waits for 'test' to pass
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'  # conditional job
    steps:
      - run: echo "Deploying..."

  # ── Step patterns ──────────────────────────────────
  examples:
    runs-on: ubuntu-latest
    steps:

      # Multi-line run command
      - name: Multi-step shell
        run: |
          echo "Step 1"
          npm install
          npm run build

      # Step with environment variable
      - name: Use secrets
        env:
          API_KEY: ${{ secrets.MY_API_KEY }}
        run: curl -H "Auth: $API_KEY" https://api.example.com

      # Continue even if step fails
      - name: Optional check
        continue-on-error: true
        run: npm audit

      # Set output for next steps
      - name: Get version
        id: pkg
        run: echo "version=$(node -p "require('./package.json').version")" >> $GITHUB_OUTPUT

      - name: Use version
        run: echo "Version is ${{ steps.pkg.outputs.version }}"
💡 CONTEXT REFERENCE
github.* — repo info, sha, ref, actor  |  env.* — workflow env vars  |  secrets.* — encrypted secrets  |  steps.ID.outputs.* — step outputs  |  needs.JOB.outputs.* — job outputs

Secrets & Environment Variables

  • Go to your GitHub repo → Settings → Secrets and variables → Actions
  • Click New repository secret, name it (e.g. DOCKER_PASSWORD), paste value
  • Reference it in YAML as ${{ secrets.DOCKER_PASSWORD }}
  • For org-wide secrets, go to Organization Settings → secrets (same flow)
using secrets YAML 
steps:
  - name: Login to Docker Hub
    uses: docker/login-action@v3
    with:
      username: ${{ secrets.DOCKER_USERNAME }}
      password: ${{ secrets.DOCKER_PASSWORD }}

  - name: Deploy to AWS
    env:
      AWS_ACCESS_KEY_ID: ${{ secrets.AWS_ACCESS_KEY_ID }}
      AWS_SECRET_ACCESS_KEY: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
      AWS_REGION: us-east-1           # non-secret env var
    run: aws s3 sync dist/ s3://my-bucket/
⚠️ SECURITY
Secrets are masked in logs automatically. Never echo or print secret values. Use GITHUB_TOKEN (auto-created) for repo operations — no manual setup needed.
06

Matrix Builds

Run the same job across multiple OS/version combinations simultaneously.

matrix strategy YAML 
jobs:
  test:
    runs-on: ${{ matrix.os }}

    strategy:
      fail-fast: false             # don't cancel all if one fails
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
        node: [18, 20, 22]
        # Exclude specific combos
        exclude:
          - os: windows-latest
            node: 18

    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node }}
      - run: npm ci && npm test
✅ TIP
The example above spawns 3×3 = 9 parallel jobs (minus 1 excluded = 8 jobs). Perfect for libraries that need cross-platform/version guarantees.

Artifacts & Caching

upload & download artifacts YAML 
steps:
  - run: npm run build          # creates ./dist folder

  # Upload build output as artifact
  - uses: actions/upload-artifact@v4
    with:
      name: dist-bundle
      path: dist/
      retention-days: 7

# In a downstream job: download artifact
- uses: actions/download-artifact@v4
  with:
    name: dist-bundle
    path: ./dist
dependency caching (faster builds) YAML 
steps:
  # Cache node_modules across runs
  - uses: actions/cache@v4
    with:
      path: ~/.npm
      key: npm-${{ runner.os }}-${{ hashFiles('**/package-lock.json') }}
      restore-keys: |
        npm-${{ runner.os }}-

  # Cache pip packages (Python)
  - uses: actions/cache@v4
    with:
      path: ~/.cache/pip
      key: pip-${{ hashFiles('requirements.txt') }}

.github/workflows/node-api-cicd.yml NODE.JS API — FULL CI/CD 
name: Node.js API CI/CD

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

env:
  IMAGE_NAME: my-node-api

jobs:

  test:
    name: Lint & Test
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20', cache: npm }
      - run: npm ci
      - run: npm run lint
      - run: npm test -- --coverage
      - uses: actions/upload-artifact@v4
        with:
          name: coverage
          path: coverage/

  docker-build:
    name: Build & Push Docker
    needs: test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    steps:
      - uses: actions/checkout@v4
      - uses: docker/login-action@v3
        with:
          username: ${{ secrets.DOCKER_USERNAME }}
          password: ${{ secrets.DOCKER_PASSWORD }}
      - name: Build and push
        uses: docker/build-push-action@v5
        with:
          push: true
          tags: ${{ secrets.DOCKER_USERNAME }}/${{ env.IMAGE_NAME }}:latest

  deploy:
    name: Deploy to EC2
    needs: docker-build
    runs-on: ubuntu-latest
    steps:
      - name: SSH deploy
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.EC2_HOST }}
          username: ubuntu
          key: ${{ secrets.EC2_SSH_KEY }}
          script: |
            docker pull ${{ secrets.DOCKER_USERNAME }}/${{ env.IMAGE_NAME }}:latest
            docker stop api || true
            docker rm api || true
            docker run -d --name api -p 3000:3000 \
              -e NODE_ENV=production \
              ${{ secrets.DOCKER_USERNAME }}/${{ env.IMAGE_NAME }}:latest
.github/workflows/python-publish.yml PYTHON — PYPI PUBLISH 
name: Test & Publish Python Package

on:
  push:
    branches: [main]
  release:
    types: [published]

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        python-version: ['3.10', '3.11', '3.12']
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: ${{ matrix.python-version }}
      - run: pip install -e ".[dev]"
      - run: pytest --tb=short

  publish:
    needs: test
    runs-on: ubuntu-latest
    if: github.event_name == 'release'
    environment: pypi
    permissions:
      id-token: write         # for OIDC trusted publishing
    steps:
      - uses: actions/checkout@v4
      - run: pip install build && python -m build
      - uses: pypa/gh-action-pypi-publish@release/v1
.github/workflows/react-deploy.yml REACT — GITHUB PAGES 
name: Deploy React to GitHub Pages

on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: { node-version: '20', cache: npm }
      - run: npm ci && npm run build
      - uses: actions/configure-pages@v4
      - uses: actions/upload-pages-artifact@v3
        with:
          path: dist/
      - id: deployment
        uses: actions/deploy-pages@v4

Quick Cheat Sheet

📁 Directory structure
your-repo/
├── .github/
│   └── workflows/
│       ├── ci.yml

│       └── nightly.yml

🔣 Context expressions
${{ github.sha }}          commit SHA
${{ github.ref_name }}     branch name
${{ github.actor }}       who triggered
${{ runner.os }}          Linux/Windows/macOS
${{ secrets.MY_SECRET }}  encrypted secret
${{ vars.MY_VAR }}        plain variable
🏃 Runner options
ubuntu-latest    Ubuntu 24.04
ubuntu-22.04     specific version
windows-latest   Windows Server 2022
macos-latest     macOS 14 (Apple M1)
self-hosted      your own machine

⚡ Conditions (if:)

# Even if previous step failed
if: always()

if: failure()
