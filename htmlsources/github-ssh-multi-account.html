How It Works — Big Picture
Each GitHub account gets its own SSH key + an alias host in config

2
Generate 3 SSH Keys — One Per Account
Run each block separately. Give a unique filename for each key.
Personal Account 
# Generate key for Personal
ssh-keygen -t rsa -b 4096 \
  -C "personal@gmail.com" \
  -f ~/.ssh/id_rsa_personal

# When prompted for passphrase:
# Press Enter (no passphrase) OR
# type a password for extra security

# This creates 2 files:
~/.ssh/id_rsa_personal      ← private key (NEVER share)
~/.ssh/id_rsa_personal.pub  ← public key (add to GitHub)
Work Account 
# Generate key for Work
ssh-keygen -t rsa -b 4096 \
  -C "work@company.com" \
  -f ~/.ssh/id_rsa_work

# When prompted for passphrase:
# Press Enter (no passphrase) OR
# type a password for extra security

# This creates 2 files:
~/.ssh/id_rsa_work          ← private key (NEVER share)
~/.ssh/id_rsa_work.pub      ← public key (add to GitHub)
Freelance Account 
# Generate key for Freelance
ssh-keygen -t rsa -b 4096 \
  -C "freelance@gmail.com" \
  -f ~/.ssh/id_rsa_freelance

# When prompted for passphrase:
# Press Enter (no passphrase) OR

Add Public Keys to Each GitHub Account
Copy the .pub key → paste into GitHub Settings. Do this for all 3 accounts.
Personal — copy key 
# Mac
cat ~/.ssh/id_rsa_personal.pub | pbcopy

# Linux
cat ~/.ssh/id_rsa_personal.pub | xclip

# Windows
type %USERPROFILE%\.ssh\id_rsa_personal.pub | clip

# Now paste into GitHub Personal account
Work — copy key 
# Mac
cat ~/.ssh/id_rsa_work.pub | pbcopy

# Linux
cat ~/.ssh/id_rsa_work.pub | xclip

# Windows
type %USERPROFILE%\.ssh\id_rsa_work.pub | clip

# Now paste into GitHub Work account
Freelance — copy key 
# Mac
cat ~/.ssh/id_rsa_freelance.pub | pbcopy

# Linux
cat ~/.ssh/id_rsa_freelance.pub | xclip

# Windows
type %USERPROFILE%\.ssh\id_rsa_freelance.pub | clip

# Now paste into GitHub Freelance account
Where to paste in GitHub — do this for EACH account 
1. Login to GitHub (the correct account)
2. Click your profile photo → Settings
3. Left sidebar → "SSH and GPG keys"
4. Click "New SSH key"
5. Title: give it a name like "My Laptop - Personal"
6. Key type: Authentication Key
7. Key: paste the copied .pub content here
8. Click "Add SSH key"

Repeat steps 1-8 for Work account and Freelance account.
4
Create the SSH Config File
This is the most important file — maps alias hosts to the right key
The config file is at ~/.ssh/config. If it doesn't exist, create it. Each Host block is an alias — e.g. github-personal instead of github.com.
open config file in editor 
# Open (or create) the SSH config file
nano ~/.ssh/config
# OR use VS Code:
code ~/.ssh/config
open config file — Windows 
# Open in Notepad (run in cmd)
notepad %USERPROFILE%\.ssh\config
# OR in VS Code:
code %USERPROFILE%\.ssh\config
~/.ssh/config — paste this entire block 
# ─────────────────────────────────────────
# GitHub SSH — 3 Accounts Config
# ─────────────────────────────────────────

# Personal Account
Host github-personal
    HostName      github.com
    User          git
    IdentityFile  ~/.ssh/id_rsa_personal
    IdentitiesOnly yes

# Work Account
Host github-work
    HostName      github.com
    User          git
    IdentityFile  ~/.ssh/id_rsa_work
    IdentitiesOnly yes

# Freelance Account
Host github-freelance
    HostName      github.com
    User          git
    IdentityFile  ~/.ssh/id_rsa_freelance
    IdentitiesOnly yes
set correct permissions on config file (Mac/Linux only) 
chmod 600 ~/.ssh/config
chmod 600 ~/.ssh/id_rsa_personal
chmod 600 ~/.ssh/id_rsa_work
chmod 600 ~/.ssh/id_rsa_freelance
# SSH refuses to work if permissions are too open
5
Add All 3 Keys to SSH Agent
SSH agent holds your keys in memory so you don't re-enter passphrase each time
terminal — add all keys 
# Start SSH agent
eval "$(ssh-agent -s)"
# should print: Agent pid 12345

# Add all 3 private keys
ssh-add ~/.ssh/id_rsa_personal
ssh-add ~/.ssh/id_rsa_work
ssh-add ~/.ssh/id_rsa_freelance

# Verify all 3 are loaded
ssh-add -l
# should list all 3 keys
To make keys persist after restart, add the ssh-add lines to your ~/.zshrc or ~/.bashrc so they load automatically every time you open a terminal.

7
Clone & Push — Use the Right Account Per Repo
Use the alias host instead of github.com in the remote URL
Key Rule: Replace github.com with your alias (github-personal, github-work, github-freelance) in the git remote URL. That's all you change.
Cloning a repo — use alias instead of github.com 
# ❌ Normal clone (uses default key — wrong account!)
git clone git@github.com:personal-user/my-repo.git

# ✅ Personal repo — use github-personal alias
git clone git@github-personal:personal-user/my-repo.git

# ✅ Work repo — use github-work alias
git clone git@github-work:work-org/work-repo.git

# ✅ Freelance repo — use github-freelance alias
git clone git@github-freelance:freelance-user/client-repo.git
Already have a repo locally? Update its remote URL 
# Check current remote URL
git remote -v

# Update to use correct account alias

# For a Personal repo
git remote set-url origin git@github-personal:personal-user/repo.git

# For a Work repo
git remote set-url origin git@github-work:work-org/repo.git

# For a Freelance repo
git remote set-url origin git@github-freelance:freelance-user/repo.git

# Verify it changed
git remote -v
Push & Pull — works normally after setting remote URL 
# Everything else is normal git — no changes needed
git add .
git commit -m "your message"
git push origin main      ← uses correct account automatically
git pull origin main      ← uses correct account automatically
