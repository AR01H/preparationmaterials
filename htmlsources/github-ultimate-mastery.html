Git Internals & Setup
How Git actually works under the hood, configuration, and SSH setup

How Git Stores Data

Git is a content-addressable filesystem. Every piece of data is stored as an object identified by its SHA-1 hash. There are 4 object types:

ObjectDescriptionContains
blobFile contentsRaw file data
treeDirectory listingBlob/tree pointers + names + permissions
commitSnapshotTree pointer, parent commit, author, message
tagNamed commitCommit pointer + tagger + message
commit abc123 │ ├─ tree def456 (root directory) │ ├─ blob 111aaa → src/index.js │ ├─ blob 222bbb → package.json │ └─ tree 333ccc → src/ │ └─ blob 444ddd → src/app.js │ ├─ parent: prev-commit-sha ├─ author: Shankar <shankar@example.com> └─ message: feat: add login

First-Time Setup

global configbash
# Identity (required — used in every commit)
git config --global user.name "Shankar"
git config --global user.email "shankar@example.com"

# Default branch
git config --global init.defaultBranch main

# Editor
git config --global core.editor "code --wait"

# Diff tool
git config --global diff.tool vscode
git config --global difftool.vscode.cmd 'code --wait --diff $LOCAL $REMOTE'

# Pull strategy (rebase instead of merge)
git config --global pull.rebase true

# Useful aliases
git config --global alias.st status
git config --global alias.co checkout
git config --global alias.lg "log --oneline --graph --all --decorate"
git config --global alias.undo "reset --soft HEAD~1"

# View all config
git config --list --show-origin

SSH Key Setup (Recommended)

SSH setup for GitHubbash
# 1. Generate ED25519 key (modern, secure, fast)
ssh-keygen -t ed25519 -C "shankar@example.com" -f ~/.ssh/github_ed25519

# 2. Add to ssh-agent
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/github_ed25519

# 3. Copy public key to clipboard
cat ~/.ssh/github_ed25519.pub    # then paste into GitHub → Settings → SSH Keys

# 4. Configure SSH to use this key for GitHub (~/.ssh/config)
Host github.com
  HostName github.com
  User git
  IdentityFile ~/.ssh/github_ed25519
  AddKeysToAgent yes

# 5. Test connection
ssh -T git@github.com
# Hi Shankar! You've successfully authenticated...

# 6. Now clone with SSH
git clone git@github.com:username/repo.git

GPG Signed Commits

verified commitsbash
# Generate GPG key
gpg --full-generate-key    # choose RSA 4096

# Get key ID
gpg --list-secret-keys --keyid-format=long

# Export public key → paste into GitHub Settings → GPG Keys
gpg --armor --export YOUR_KEY_ID

# Tell git to use it
git config --global user.signingkey YOUR_KEY_ID
git config --global commit.gpgsign true    # sign all commits

# Commits now show "Verified" badge on GitHub

Advanced Git Commands

power commandsbash
# Stash work-in-progress
git stash push -m "WIP: login feature"
git stash list
git stash pop                       # restore latest stash
git stash apply stash@{2}           # restore specific stash

# Find which commit broke something
git bisect start
git bisect bad                      # current commit is bad
git bisect good v1.0.0              # this tag was good
# git tests each commit, you mark good/bad until culprit found
git bisect reset

# See who changed each line
git blame -L 10,20 src/index.js
git blame -w --ignore-rev abc123 src/index.js   # ignore reformatting commit

# Find a deleted file
git log --all --full-history -- path/to/file.js

# Recover a deleted branch
git reflog                          # find the last commit SHA
git checkout -b recovered-branch SHA

# Clean untracked files
git clean -fd                       # remove untracked files and dirs
git clean -fdn                      # dry run (preview)

# Partial commit (stage specific lines)
git add -p file.js                  # interactive hunk staging

# Rewrite author on last N commits
git rebase -i HEAD~3

GitHub Pages
Host static websites, documentation, and portfolios directly from your repo — for free

What is GitHub Pages?

GitHub Pages serves static files (HTML, CSS, JS) directly from your repository at https://username.github.io/repo-name or a custom domain — completely free.

TypeURLSource
User/Org siteusername.github.ioRepo named username.github.io
Project siteusername.github.io/repoAny repo, docs/ or gh-pages branch
Custom domainyourdomain.comAdd CNAME file + DNS config

Enable GitHub Pages

1

Go to Settings → Pages

Repository → Settings → Pages

2

Choose source

Branch: main or gh-pages, Folder: / (root) or /docs

3

Site goes live

Available at https://username.github.io/repo within ~1 minute

Deploy with GitHub Actions (Recommended)

.github/workflows/pages.ymlyaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: {node-version: '20', cache: 'npm'}
      - run: npm ci && npm run build
      - name: Setup Pages
        uses: actions/configure-pages@v4
      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: ./dist

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - id: deployment
        uses: actions/deploy-pages@v4

Push and pull Docker images at ghcr.io/owner/image-name. Integrates natively with GITHUB_TOKEN.

name: Publish Docker Image
on:
  push:
    branches: [main]
    tags: ['v*']

permissions:
  contents: read
  packages: write

jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Log in to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Build and push
        run: |
          IMAGE=ghcr.io/${{ github.repository_owner }}/my-app
          docker build -t $IMAGE:latest -t $IMAGE:${{ github.sha }} .
          docker push $IMAGE --all-tags

Webhooks
Trigger your own servers when GitHub events happen — build integrations and bots

What are Webhooks?

Webhooks send an HTTP POST request to your server whenever specific events occur on GitHub. This lets you build custom integrations without polling the API.

Creating a Webhook

1

Go to Settings → Webhooks → Add webhook

In your repo or organization settings

2

Enter your Payload URL

https://yourserver.com/github/webhook

3

Set a Secret

A random string used to verify the payload signature. Store this as a server env var.

4

Select events

Choose specific events or "Send me everything"

Handling Webhooks in Node.js

webhook receiver (Express)javascript
const express = require('express');
const crypto = require('crypto');

const app = express();
app.use(express.json());

function verifySignature(req) {
  const signature = req.headers['x-hub-signature-256'];
  const hmac = crypto.createHmac('sha256', process.env.WEBHOOK_SECRET);
  const digest = 'sha256=' + hmac.update(JSON.stringify(req.body)).digest('hex');
  return crypto.timingSafeEqual(Buffer.from(signature), Buffer.from(digest));
}

app.post('/github/webhook', (req, res) => {
  if (!verifySignature(req)) {
    return res.status(401).send('Unauthorized');
  }

  const event = req.headers['x-github-event'];
  const payload = req.body;

  switch (event) {
    case 'push':
      console.log(`Push to ${payload.ref} by ${payload.pusher.name}`);
      break;
    case 'pull_request':
      if (payload.action === 'opened') {
        console.log(`New PR: ${payload.pull_request.title}`);
        // Trigger your CI, notify Slack, etc.
      }
      break;
  }

  res.status(200).send('OK');
});

app.listen(3000);
🚫
Always Verify the Signature
Anyone can POST to your webhook URL. Always verify the X-Hub-Signature-256 header using HMAC-SHA256 with your webhook secret to confirm the request is genuinely from GitHub.

Workflow File Anatomy

.github/workflows/ci.yml ├── name: → Display name in GitHub UI ├── on: → Events that trigger this workflow ├── env: → Workflow-level environment variables ├── permissions: → GITHUB_TOKEN permission scope ├── concurrency: → Cancel/queue duplicate runs └── jobs: ├── job-id: │ ├── name: → Display name │ ├── runs-on: → Runner OS │ ├── needs: → Dependency on other jobs │ ├── if: → Conditional execution │ ├── environment: → Deployment environment │ ├── env: → Job-level vars │ ├── outputs: → Expose values to other jobs │ └── steps: │ ├── uses: → Run a reusable Action │ ├── run: → Run shell commands │ ├── with: → Pass inputs to an Action │ ├── env: → Step-level vars │ └── if: → Conditional step

on:

    branches: [main]

# Restrict default token permissions

  contents: read

# Don't run duplicate workflows for same branch

env:

      - name: Checkout

          node-version: ${{ env.NODE_VERSION }}

      - run: npm test

          path: test-results/

Workflow Triggers (on:)
Every way to trigger a workflow — events, filters, schedules, and manual dispatch

Complete Trigger Reference

all triggersyaml
on:
  # Push with branch/path/tag filters
  push:
    branches: [main, 'release/**']
    branches-ignore: ['dependabot/**']
    paths: ['src/**', 'package.json']
    paths-ignore: ['**.md']
    tags: ['v*']

  # Pull request events
  pull_request:
    types: [opened, synchronize, reopened, ready_for_review, labeled]
    branches: [main]

  # PR from fork — has write permissions (use carefully!)
  pull_request_target:
    types: [opened]

  # Cron schedule (UTC)
  schedule:
    - cron: '0 9 * * 1-5'    # Mon-Fri 9 AM UTC
    - cron: '0 0 1 * *'     # 1st of every month

  # Manual trigger with inputs
  workflow_dispatch:
    inputs:
      environment:
        type: choice
        options: [staging, production]
        required: true
      version:
        type: string
      dry-run:
        type: boolean
        default: false

  # Called by another workflow
  workflow_call:
    inputs:
      version: {type: string, required: true}
    secrets:
      deploy-token: {required: true}

  # External trigger (repository_dispatch)
  repository_dispatch:
    types: [deploy, rebuild]

  # Other events
  release:
    types: [published, prereleased]
  issues:
    types: [opened, labeled]
  issue_comment:
    types: [created]
  create:          # branch or tag created
  delete:          # branch or tag deleted
  fork:
  star:
    types: [created]

Job Dependency Graph

complex job dependenciesyaml
jobs:
  lint:                          # ┐ run in parallel
    runs-on: ubuntu-latest       # │
    steps: [...]                  # │
                                   # │
  test:                          # ┘ run in parallel

      version: ${{ steps.v.outputs.version }}

      - id: v

    runs-on: ubuntu-latest
    steps:
      - run: echo "v${{ needs.test.outputs.version }}"

    environment:
      name: production
      url: https://example.com
    steps: [...]

  # Always run — even if build fails
  notify:
    needs: [build, deploy]
    if: always()
    runs-on: ubuntu-latest
    steps:
      - run: echo "Build: ${{ needs.build.result }}, Deploy: ${{ needs.deploy.result }}"

src/index.js — full examplejavascript
const core = require('@actions/core');
const github = require('@actions/github');
const exec = require('@actions/exec');
const io = require('@actions/io');

async function run() {
  try {
    // ── Inputs ────────────────────────────────────────
    const token = core.getInput('token', { required: true });
    const message = core.getInput('message');
    const failOnError = core.getBooleanInput('fail-on-error');

    // ── GitHub API ────────────────────────────────────
    const octokit = github.getOctokit(token);
    const { owner, repo } = github.context.repo;

    // Create a comment on the PR
    if (github.context.payload.pull_request) {
      await octokit.rest.issues.createComment({
        owner, repo,
        issue_number: github.context.payload.pull_request.number,
        body: message
      });
    }

    // ── Run commands ──────────────────────────────────
    let output = '';
    await exec.exec('npm', ['audit', '--json'], {
      listeners: { stdout: (data) => { output += data.toString(); }}
    });

    // ── Outputs ───────────────────────────────────────
    core.setOutput('result', 'success');

    // ── Debug / Logging ───────────────────────────────
    core.debug('Debug message (only shown with ACTIONS_STEP_DEBUG=true)');
    core.info('Info message');
    core.warning('Warning — shows in annotations');

    // ── Masking secrets ───────────────────────────────
    core.setSecret(token);  // masks this value in all subsequent logs

    // ── Groups in logs ───────────────────────────────
    core.startGroup('Detailed output');
    console.log(output);
    core.endGroup();

  } catch (error) {
    core.setFailed(error.message);
  }
}

Secret Scanning

GitHub automatically scans all commits for known secret patterns (AWS keys, Stripe keys, GitHub tokens, etc.) and alerts you immediately.

Enable

Repo → Settings → Security → Secret scanning → Enable

2

Push protection

Blocks pushes containing secrets before they reach the repo. Enable in the same settings page.

Custom patterns

🚨
If a secret IS committed
1. Revoke the secret immediately — even before removing it from history. 2. Use git filter-repo (not BFG) to remove from history. 3. Force-push all branches. 4. Generate a new secret. Assume it is compromised from the moment it was committed.

Building a Simple GitHub App (Node.js)

app.js using @octokit/appjavascript
import { App, createNodeMiddleware } from "@octokit/app";
import { createServer } from "http";

const app = new App({
  appId: process.env.APP_ID,
  privateKey: process.env.PRIVATE_KEY,
  webhooks: { secret: process.env.WEBHOOK_SECRET },
});

// Auto-label new issues
app.webhooks.on("issues.opened", async ({ octokit, payload }) => {
  await octokit.rest.issues.addLabels({
    owner: payload.repository.owner.login,
    repo: payload.repository.name,
    issue_number: payload.issue.number,
    labels: ["triage"]
  });
});

// Auto-approve Dependabot PRs
app.webhooks.on("pull_request.opened", async ({ octokit, payload }) => {
  if (payload.sender.login === "dependabot[bot]") {
    await octokit.rest.pulls.createReview({
      owner: payload.repository.owner.login,
      repo: payload.repository.name,
      pull_number: payload.pull_request.number,
      event: "APPROVE"
    });
  }
});

const server = createServer(createNodeMiddleware(app));
server.listen(3000);

OAuth Apps
Let users log in with GitHub — implement OAuth 2.0 in your web applications

OAuth Flow

User clicks "Login with GitHub" │ ▼ Your app redirects to GitHub authorization: https://github.com/login/oauth/authorize ?client_id=YOUR_CLIENT_ID &redirect_uri=https://yourapp.com/callback &scope=read:user,user:email &state=RANDOM_STRING ← CSRF protection │ ▼ (User approves) GitHub redirects back to your callback URL: https://yourapp.com/callback?code=TEMP_CODE&state=RANDOM_STRING │ ▼ (Backend only — never expose client_secret!) POST https://github.com/login/oauth/access_token client_id + client_secret + code │ ▼ Receive: access_token + scope + token_type │ ▼ Use token to call GitHub API as the user

OAuth Implementation (Node.js/Express)

oauth-server.jsjavascript
const express = require('express'); const crypto = require('crypto'); const CLIENT_ID = process.env.GITHUB_CLIENT_ID; const CLIENT_SECRET = process.env.GITHUB_CLIENT_SECRET; const stateStore = new Map(); // Step 1: Redirect to GitHub app.get('/auth/github', (req, res) => { const state = crypto.randomBytes(16).toString('hex'); stateStore.set(state, true); const params = new URLSearchParams({ client_id: CLIENT_ID, redirect_uri: 'https://yourapp.com/auth/callback', scope: 'read:user user:email', state }); res.redirect(`https://github.com/login/oauth/authorize?${params}`); }); // Step 2: Handle callback app.get('/auth/callback', async (req, res) => { const { code, state } = req.query; // Validate state (CSRF protection) if (!stateStore.has(state)) return res.status(403).send('Invalid state'); stateStore.delete(state); // Exchange code for token (server-side only) const tokenRes = await fetch('https://github.com/login/oauth/access_token', { method: 'POST', headers: { 'Accept': 'application/json', 'Content-Type': 'application/json' }, body: JSON.stringify({ client_id: CLIENT_ID, client_secret: CLIENT_SECRET, code }) }); const { access_token } = await tokenRes.json(); // Get user info const userRes = await fetch('https://api.github.com/user', { headers: { 'Authorization': `token ${access_token}` } }); const user = await userRes.json(); // Store in session, redirect to app req.session.user = { id: user.id, login: user.login, name: user.name }; res.redirect('/dashboard'); });

Self-Hosted Runners
Run workflows on your own machines for full control, no billing, and custom environments

When to Use Self-Hosted Runners

  • Cost — No GitHub billing minutes for private repos
  • Hardware — Need specific CPU/GPU/RAM not available on GitHub-hosted runners
  • Network — Need access to private VPC resources (databases, internal APIs)
  • Compliance — Code must not leave your infrastructure
  • Caching — Local cache persists across runs (faster builds)

Setting Up a Self-Hosted Runner

1

Register

Repo → Settings → Actions → Runners → New self-hosted runner → Choose OS

2

Install & configure on your machine

bash
# Download runner (version from GitHub UI)
mkdir actions-runner && cd actions-runner
curl -o actions-runner-linux-x64.tar.gz -L \
  https://github.com/actions/runner/releases/download/v2.x.x/actions-runner-linux-x64-2.x.x.tar.gz
tar xzf ./actions-runner-linux-x64.tar.gz

# Configure (use token from GitHub UI)
./config.sh --url https://github.com/your-org/your-repo \
  --token YOUR_REGISTRATION_TOKEN \
  --name "my-runner" \
  --labels "self-hosted,linux,x64,gpu"

# Run as a service (recommended)
sudo ./svc.sh install
sudo ./svc.sh start
3

Use in workflow

yaml
jobs:
  build:
    runs-on: [self-hosted, linux, gpu]  # match your labels
    steps:
      - run: nvidia-smi  # GPU available!

Runner Groups (Organization)

Organization runners can be grouped and assigned to specific repositories:

  • Default group — all repos can use
  • Custom groups — restrict to specific repos (e.g., only production-deploy repo can use prod-runners)
  • Configure at: Org Settings → Actions → Runner groups
⚠️
Security Warning for Public Repos
Never use self-hosted runners for public repositories — any fork PR can execute arbitrary code on your machine. Self-hosted runners should only be used for private repos or organization repos with proper access controls.

Complete Mono-Repo CI/CD

monorepo-pipeline.ymlyaml
name: Mono-repo CI/CD

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

permissions:
  contents: read
  pull-requests: write

jobs:
  # Detect what changed to only run relevant jobs
  changes:
    runs-on: ubuntu-latest
    outputs:
      api: ${{ steps.filter.outputs.api }}
      web: ${{ steps.filter.outputs.web }}
      mobile: ${{ steps.filter.outputs.mobile }}
    steps:
      - uses: actions/checkout@v4
      - id: filter
        uses: dorny/paths-filter@v3
        with:
          filters: |
            api:   ['packages/api/**']
            web:   ['packages/web/**']
            mobile: ['packages/mobile/**']

  api-tests:
    needs: changes
    if: needs.changes.outputs.api == 'true'
    runs-on: ubuntu-latest
    services:
      postgres:
        image: postgres:16
        env: {POSTGRES_DB: test, POSTGRES_USER: test, POSTGRES_PASSWORD: test}
        ports: ['5432:5432']
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: {node-version: '20', cache: 'npm'}
      - run: npm ci && npm test --workspace=packages/api
        env:
          DATABASE_URL: postgresql://test:test@localhost:5432/test

  web-tests:
    needs: changes
    if: needs.changes.outputs.web == 'true'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with: {node-version: '20', cache: 'npm'}
      - run: npm ci && npm test --workspace=packages/web

  deploy:
    needs: [api-tests, web-tests]
    if: |
      always() &&
      github.ref == 'refs/heads/main' &&
      !contains(needs.*.result, 'failure')
    runs-on: ubuntu-latest
    permissions:
      id-token: write
      contents: read
    steps:
      - run: echo "Deploying changed packages..."
