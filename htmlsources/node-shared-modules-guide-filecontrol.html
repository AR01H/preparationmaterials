Install NVM + All Node Versions
NVM lets you switch between Node 16 / 18 / 22 anytime
terminal — Mac / Linux 
# Step 2a — Install NVM
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

# Step 2b — Reload terminal config
source ~/.zshrc
# OR if using bash:
source ~/.bashrc

# Step 2c — Install all 3 Node versions
nvm install 16
nvm install 18
nvm install 22

# Step 2d — Verify all installed
nvm list
# should show →  v16.x.x  v18.x.x  v22.x.x
cmd / powershell — Windows 
# Step 2a — Download nvm-windows installer from:
https://github.com/coreybutler/nvm-windows/releases
# → download nvm-setup.exe → run it

# Step 2b — Open NEW cmd window, then install Node versions
nvm install 16
nvm install 18
nvm install 22

# Step 2c — Verify
nvm list

# Step 2d — Set NODE_PATH in System Environment Variables
# Go to: Control Panel → System → Advanced → Environment Variables
# Add new variable:
Variable name:  NODE_PATH
Variable value: C:\Users\YOU\shared_modules\node16\node_modules
# (change per project when switching)

Add Your Own Custom File (my-utils)
Drop your own shared functions — use like any npm package in all projects
Create my-utils folder inside each shared node_modules. Then just import from 'my-utils' in any project — no path needed!
terminal — create folders 
# create my-utils in all 3
mkdir ~/shared_modules/node16\
  /node_modules/my-utils

mkdir ~/shared_modules/node18\
  /node_modules/my-utils

mkdir ~/shared_modules/node22\
  /node_modules/my-utils
my-utils/package.json 
{
  "name": "my-utils",
  "version": "1.0.0",
  "main": "index.js"
}

# Copy this file into:
# node16/node_modules/my-utils/
# node18/node_modules/my-utils/
# node22/node_modules/my-utils/
my-utils/index.js 
// Shared constants
export const API_URL =
  "https://api.mycompany.com";

// Shared utility functions
export const formatDate = (d) =>
  new Date(d).toLocaleDateString();

export const formatCurrency = (n) =>
  "₹" + n.toLocaleString("en-IN");

export const isEmpty = (v) =>
  v === null || v === "";
How to import in ANY project — no path, no install 
// shop-app/src/App.js  OR  admin-panel/src/App.js — same import everywhere
import { API_URL, formatDate, formatCurrency } from 'my-utils';
import axios from 'axios';      // from shared
import React from 'react';      // from shared

// Use directly
console.log(formatDate(new Date()));   // 4/19/2026
console.log(formatCurrency(49999));    // ₹49,999
console.log(API_URL);                   // https://api.mycompany.com
5
Set NODE_PATH Inside Project — Pick ONE Way
Both ways keep the path inside your project code. Choose whichever fits you — then stick to it.
OR
Way 1 — cross-env in package.json: You add cross-env NODE_PATH=../shared_modules/nodeXX/node_modules before every script command. cross-env is the only package installed locally — it sets the path at runtime before Node starts.
shop-app/package.json — Node 16 
{
  "name": "shop-app",
  "scripts": {
    "start": "cross-env NODE_PATH=../shared_modules/node16/node_modules react-scripts start",
    "build": "cross-env NODE_PATH=../shared_modules/node16/node_modules react-scripts build",
    "test":  "cross-env NODE_PATH=../shared_modules/node16/node_modules react-scripts test"
  },
  "devDependencies": {
    "cross-env": "^7.0.3"   ← only this one local install
  }
}

# dashboard-app/package.json → same, node16
# admin-panel/package.json  → same, node18
# landing-page/package.json → same, node22
terminal — install cross-env in each project 
# Run once per project — installs cross-env locally
cd ~/projects/shop-app     && npm install --save-dev cross-env
cd ~/projects/dashboard-app && npm install --save-dev cross-env
cd ~/projects/admin-panel   && npm install --save-dev cross-env
cd ~/projects/landing-page  && npm install --save-dev cross-env

# That's it. Now just run as normal:
npm start      ✅ NODE_PATH injected automatically before start
npm run build  ✅ same for build
npm test       ✅ same for test
Relative vs Absolute path in the script 
# Relative (recommended — projects & shared_modules side by side)
NODE_PATH=../shared_modules/node16/node_modules

# Absolute — Mac/Linux (if folder is somewhere else)
NODE_PATH=/home/yourname/shared_modules/node16/node_modules

# Absolute — Windows
NODE_PATH=C:/Users/YourName/shared_modules/node16/node_modules
jsconfig.json — VS Code IntelliSense (add to each project) 
// shop-app/jsconfig.json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "my-utils": ["../shared_modules/node16/node_modules/my-utils"],
      "*": ["../shared_modules/node16/node_modules/*"]
    }
  }
}
// Gives autocomplete + go-to-definition in VS Code
Path is relative to where you run npm. If projects/ and shared_modules/ are both under ~/, then ../shared_modules/node16/node_modules always resolves correctly. If your layout is different, switch to an absolute path.
Way 2 — .env file in project root: You create a .env file inside each project with NODE_PATH= set. react-scripts (Create React App) automatically reads .env before starting. No cross-env, no changes to package.json scripts at all.
shop-app/.env & dashboard-app/.env 
# shop-app/.env
NODE_PATH=../shared_modules/node16/node_modules

# That's the entire file — one line only.
# react-scripts reads this automatically
# before npm start / build / test.
admin-panel/.env 
# admin-panel/.env
NODE_PATH=../shared_modules/node18/node_modules

# landing-page/.env
NODE_PATH=../shared_modules/node22/node_modules
package.json — stays completely clean 
{
  "name": "shop-app",
  "scripts": {
    "start": "react-scripts start",
    "build": "react-scripts build",
    "test":  "react-scripts test"
  }
}
// No cross-env, no NODE_PATH here.
// It's all in .env — package.json untouched.
terminal — create .env files for all 4 projects at once 
# Create .env file in each project — one command per project
echo "NODE_PATH=../shared_modules/node16/node_modules" > ~/projects/shop-app/.env
echo "NODE_PATH=../shared_modules/node16/node_modules" > ~/projects/dashboard-app/.env
echo "NODE_PATH=../shared_modules/node18/node_modules" > ~/projects/admin-panel/.env
echo "NODE_PATH=../shared_modules/node22/node_modules" > ~/projects/landing-page/.env

# Verify:
cat ~/projects/shop-app/.env
# → NODE_PATH=../shared_modules/node16/node_modules  ✅

# Now just run as usual — .env is picked up automatically:
npm start      ✅
npm run build  ✅

jsconfig.json — VS Code IntelliSense (add to each project) 
// shop-app/jsconfig.json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "my-utils": ["../shared_modules/node16/node_modules/my-utils"],

Works only with react-scripts (Create React App). If you use Vite, Next.js, or plain Node — use Way 1 (cross-env) instead, as those tools don't auto-read .env for NODE_PATH. Also add .env to .gitignore if the path is machine-specific.

# Add .nvmrc to each project (tells nvm which node version to use)
echo "16" > ~/projects/shop-app/.nvmrc

Daily Usage — How to Work
This is what you do every day from now on
terminal — switching between projects 
# Working on shop-app (Node 16)
cd ~/projects/shop-app
nvm use        # reads .nvmrc → switches to node 16
npm start
# cross-env sets NODE_PATH → ../shared_modules/node16/node_modules
✅ works!

# Switch to admin-panel (Node 18)
cd ~/projects/admin-panel
nvm use        # reads .nvmrc → switches to node 18
npm start
# cross-env sets NODE_PATH → ../shared_modules/node18/node_modules
✅ works!

# Switch to landing-page (Node 22)
cd ~/projects/landing-page
nvm use        # reads .nvmrc → switches to node 22
npm start
# cross-env sets NODE_PATH → ../shared_modules/node22/node_modules
✅ works!
Adding a new package later 
# Need to add a new package?
# Install into shared folder, NOT in project

# Example: add moment.js for node16 projects
nvm use 16
cd ~/shared_modules/node16
npm install moment

# Now ALL node16 projects can use it.
# Their package.json already points here via NODE_PATH.
# In shop-app or dashboard-app:
import moment from 'moment';   ✅

# Update my-utils? Just edit the file:
nano ~/shared_modules/node16\
  /node_modules/my-utils/index.js
# All projects get the update instantly —
# no reinstall, no restart needed!
Never run npm install <package> inside a project folder. Always install into ~/shared_modules/nodeXX/. The only thing installed locally in each project is cross-env (devDependency) which is what reads the path from package.json.
